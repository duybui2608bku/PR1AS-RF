1 Tổng quan dự án

PR1AS là nền tảng marketplace kết nối khách hàng (Client) với worker cung cấp các dịch vụ đồng hành và hỗ trợ. Hệ thống hỗ trợ toàn bộ vòng đời từ tìm kiếm worker, đặt lịch, thanh toán, đến đánh giá sau khi hoàn thành dịch vụ.

2 Hệ thống Real-time

Vai trò trong hệ thống

👤 Client

Khách hàng tìm kiếm và đặt dịch vụ từ worker. Quản lý ví, lịch sử booking, và đánh giá worker sau khi hoàn thành.

🛠️ Worker

Người cung cấp dịch vụ. Cấu hình hồ sơ, danh sách dịch vụ và bảng giá. Nhận và quản lý các booking từ client.

🔧 Admin

Quản trị viên hệ thống. Giám sát giao dịch, xử lý tranh chấp, quản lý người dùng, cấu hình danh tiếng và cài đặt site.

Phạm vi dịch vụ

🤝 Companionship Services

Dịch vụ đồng hành: cà phê, ăn tối, đi chơi, v.v. Có phân cấp companionship (1–3), quy tắc về tiếp xúc vật lý và trang phục.

🙋 Assistance Services

Dịch vụ hỗ trợ: hướng dẫn, phiên dịch, tư vấn, v.v. Tập trung vào kỹ năng trí tuệ và chuyên môn.

4 Kiến trúc Backend

Layered Architecture

🌐
Routes
Định nghĩa endpoint, áp dụng middleware xác thực và validation trước khi tới controller
🛡️
Middleware
authenticate, authorize, csrfProtection, rateLimiter, validateRequest (Zod)
🎮
Controllers
HTTP layer duy nhất. Dùng asyncHandler, trả về R.success() / R.error(). Không chứa business logic
⚙️
Services
Toàn bộ business logic. Không import gì từ HTTP layer. Giao tiếp với Repository
🗃️
Repositories
Toàn bộ Mongoose query. Service không bao giờ gọi Mongoose trực tiếp. Tách biệt data access
🍃
Models
Mongoose schemas + TypeScript interfaces. 19 collections trong MongoDB

Scheduled Jobs

⏰ Booking Expiration Job

Chạy qua node-cron, khởi động trong index.ts. Tự động hủy các booking ở trạng thái PENDING quá hạn xác nhận. Sử dụng JobLock collection để distributed locking, tránh race condition khi scale.

MongoDB với Mongoose ODM. Tất cả collections được đánh index tối ưu cho các query phổ biến.

Schema nổi bật: Booking

{
  client_id, worker_id, worker_service_id, service_id,
  schedule:  { start_time, end_time, duration_hours },
  pricing:   { unit: HOUR|DAY|MONTH, quantity, total_amount },
  status:    PENDING → CONFIRMED → IN_PROGRESS → COMPLETED
             | CANCELLED | DISPUTED,
  cancellation: { cancelled_by: CLIENT|WORKER, reason, notes },
  dispute:      { reason, description, evidence_urls[], disputed_by, resolution }
  // unique index: (worker_id, schedule.start_time) cho active bookings
}

10 Hệ thống Booking

Vòng đời Booking

PENDING
→
CONFIRMED
→
IN_PROGRESS
→
COMPLETED
CANCELLED
·
DISPUTED
📅 Constraint chống xung đột lịch

Unique index compound (worker_id, schedule.start_time) trên active bookings đảm bảo không có 2 booking cùng một worker tại cùng thời điểm. Expiration job tự động hủy PENDING quá hạn.

⚖️ Quy trình tranh chấp

Client hoặc Worker có thể mở dispute khi booking ở IN_PROGRESS hoặc COMPLETED. Admin xử lý qua dashboard, có group chat riêng cho mỗi tranh chấp với tất cả các bên.

Review sau Booking

Sau khi booking COMPLETED, client được phép để lại review. Rating 5 sao với 4 tiêu chí chi tiết: Chuyên nghiệp, Đúng giờ, Giao tiếp, Chất lượng dịch vụ. Worker có thể trả lời review. Review có thể bị report và ẩn.

12 Hệ thống Chat & Thông báo

Chat System

Socket.IO events: Tin nhắn được gửi qua Socket.IO để real-time delivery. Có fallback HTTP API cho lịch sử tin nhắn.

Message types: text, image, video, audio, file

Features: Read receipts, unread count, message pagination, reply-to, group chat cho tranh chấp.

Notification System

Kênh delivery:

  • 📱 In-app — Socket.IO real-time
  • 📧 Email — Nodemailer (Gmail)
  • 🔔 Push — Web Push API + browser subscription

Chống trùng lặp: dedupe_key trên mỗi notification.

Preferences: User tắt/bật từng kênh và mute từng loại thông báo.

13 Tính năng Mạng xã hội

📝 Posts

Feed bài đăng với visibility PUBLIC/PRIVATE. Tối đa 5000 ký tự. Tác giả có thể khóa bình luận. Soft delete với audit trail.

💬 Comments

Bình luận nested (parent_comment_id). Hiển thị dạng cây. Hỗ trợ reaction trên từng comment. Soft delete.

😊 Reactions

Emoji reaction trên post và comment. Unique constraint (1 user, 1 target, 1 reaction). Toggle reaction.

Hashtag System

Hashtag tự động extract từ nội dung post. usage_count tracking. API trending hashtag. Hashtag search cho discovery.
